Phase 1: Local Development & Verification
  Prerequisite: Ensure Docker and Docker Compose are installed.

   1. Configure Local Environment:
       * Create a .env file in the root directory.
       * Add your GEMINI_API_KEY=your_key_here.
   2. Verify Microservices:
       * Run docker compose up -d --build.
       * Check service health: docker compose ps.
       * Initialize the database: docker compose exec backend python seed.py.
   3. Test the App:
       * Open http://localhost:8501. Log in with testuser / password123.

  ---

  Phase 2: Infrastructure Provisioning (Terraform)
  Prerequisite: Install Terraform and AWS CLI. Configure AWS credentials (aws configure).

   1. Create an AWS Key Pair:
       * Go to AWS Console -> EC2 -> Key Pairs.
       * Create a key named stock-screener-key and download the .pem file.
   2. Initialize & Deploy:
   1     cd terraform
   2     terraform init
   3     terraform apply -var="key_name=stock-screener-key"
   3. Capture Output: Terraform will create 4 EC2 instances. Note their Public IPs from the AWS console.

  ---

  Phase 3: Self-Managed Kubernetes Cluster Setup
  Repeat these steps for the Master and 2 Worker nodes.

   1. SSH into Instances:
   1     ssh -i "stock-screener-key.pem" ubuntu@<INSTANCE_PUBLIC_IP>
   2. Run Setup Script:
       * Copy the content of scripts/setup_k8s.sh to each instance.
       * Execute: chmod +x setup_k8s.sh && sudo ./setup_k8s.sh.
   3. Initialize Master Node (Only on k8s-master):
   1     sudo kubeadm init --pod-network-cidr=192.168.0.0/16
   2     # Follow the output instructions to set up local kubeconfig:
   3     mkdir -p $HOME/.kube
   4     sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
   5     sudo chown $(id -u):$(id -g) $HOME/.kube/config
   6     # Install Calico Networking
   7     kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.28.0/manifests/calico.yaml
   4. Join Worker Nodes:
       * Copy the kubeadm join ... command generated by the Master node.
       * Run it on both Worker nodes with sudo.

  ---

  Phase 4: CI/CD Pipeline (Jenkins)

   1. Install Jenkins:
       * SSH into the jenkins-server instance.
       * Install Java and Jenkins.
       * Crucial: Install Docker and kubectl on this server.
   2. Configure Jenkins UI (http://<JENKINS_IP>:8080):
       * Install Plugins: Docker Pipeline, Kubernetes CLI, Git.
       * Add Credentials:
           * docker-hub-credentials: Your Docker Hub username/password.
           * k8s-kubeconfig: Upload the /etc/kubernetes/admin.conf file from your Master node.
   3. Create Pipeline:
       * Create a "New Item" -> Pipeline.
       * Point it to your Git repository where the Jenkinsfile resides.
       * Run the build.

  ---

  Phase 5: Deploying the Application

   1. Create Secrets:
       * Update k8s/secrets.yaml with your real Gemini API key.
       * On the Master node, run: kubectl apply -f k8s/secrets.yaml.
   2. Deploy Manifests:
       * The Jenkins pipeline will automatically run kubectl apply -f k8s/.
       * Manually check status: kubectl get pods -w.
   3. Access the Load Balancer:
       * Your Frontend is exposed via NodePort 30501.
       * Access it via: http://<ANY_WORKER_IP>:30501.

  ---

  Phase 6: Monitoring (Prometheus & Grafana)

   1. Install Helm (on the Master node):
   1     curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
   2. Deploy Monitoring Stack:
   1     helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
   2     helm repo update
   3     helm install monitoring prometheus-community/kube-prometheus-stack
   3. Access Grafana:
       * Get the admin password: 
          kubectl get secret --namespace default monitoring-grafana -o jsonpath="{.data.admin-password}" | base64
  --decode ; echo
       * Expose Grafana via NodePort or port-forwarding:
          kubectl port-forward deployment/monitoring-grafana 3000:3000 --address 0.0.0.0
       * Visit http://<MASTER_IP>:3000.
